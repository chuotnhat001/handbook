Hệ Sinh Thái Cloud Native

Container, Kubernetes, Service Mesh, Registry, Monitoring, Message Queue, Database

Nguồn: CNCF Landscape, CNCF Definition

Mục Tiêu Học Tập

Sau khi hoàn thành chương này, bạn sẽ: Hiểu bức tranh tổng thể hệ sinh thái Cloud Native — các thành phần chính và vai trò của chúng. Biết cách các components kết nối với nhau để tạo thành một platform hoàn chỉnh. Nắm được CNCF Landscape và cách navigate nó.

CNCF Landscape — Bản Đồ Toàn Cảnh

Cloud Native Computing Foundation (CNCF) duy trì một landscape chứa 1000+ projects và products, được tổ chức theo categories. Đây là "bản đồ" giúp navigate hệ sinh thái.

Các layers chính

LayerVai tròVí dụ
App Definition & DevelopmentTools xây dựng ứng dụngHelm, Backstage, Dapr
Orchestration & ManagementQuản lý containers và servicesKubernetes, Crossplane
RuntimeChạy containerscontainerd, CRI-O, gVisor
ProvisioningChuẩn bị infrastructureTerraform, Ansible, Pulumi
Observability & AnalysisMonitor và debugPrometheus, Grafana, Jaeger

Graduation levels

  • Graduated — production-ready, widely adopted (Kubernetes, Prometheus, Envoy, Helm)
  • Incubating — growing adoption, not yet proven at scale everywhere (Argo, Dapr, Backstage)
  • Sandbox — early stage, experimental

Container Runtime

Khái niệm

Container runtime là phần mềm chịu trách nhiệm chạy containers. Có 2 tầng:

  • High-level runtime (CRI) — quản lý container lifecycle, images, networking
    • containerd — default cho Docker và Kubernetes, nhẹ, production-proven
    • CRI-O — optimized cho Kubernetes, default trên OpenShift/OKD
  • Low-level runtime (OCI) — thực thi container process
    • runc — reference implementation, dùng Linux namespaces + cgroups
    • crun — viết bằng C, nhanh hơn runc
    • gVisor — sandbox runtime (Google), security-focused
    • Kata Containers — micro-VMs, isolation mạnh hơn
Kubernetes → CRI (containerd/CRI-O) → OCI (runc) → Linux Kernel

Container Orchestration

Tại sao cần orchestration

Khi có 10-100+ containers, cần hệ thống tự động: scheduling, scaling, networking, health checking, service discovery. Kubernetes là de facto standard.

Kubernetes ecosystem

ComponentVai trò
KubernetesCore orchestrator — scheduling, scaling, self-healing
HelmPackage manager — templates cho K8s manifests
KustomizeConfig management — overlay-based customization
OperatorsCustom controllers — quản lý stateful apps (databases, Kafka)
OKD/OpenShiftEnterprise K8s distribution — thêm security, CI/CD, console

Managed Kubernetes

  • EKS (AWS), AKS (Azure), GKE (Google) — cloud-managed control plane
  • OKD/OpenShift — self-managed hoặc managed (ROSA, ARO)

Service Mesh

Khái niệm

Service Mesh là infrastructure layer xử lý communication giữa services — transparent cho application code. Giải quyết: load balancing, encryption, observability, retries, circuit breaking.

Cách hoạt động — Sidecar Pattern

Service A → Proxy (Envoy) → Network → Proxy (Envoy) → Service B

Mỗi service có một sidecar proxy (thường Envoy) — tất cả traffic đi qua proxy. Control plane quản lý configuration cho tất cả proxies.

Giải pháp phổ biến

Service MeshĐặc điểm
IstioFeature-rich nhất, complex, dùng Envoy sidecar
LinkerdLightweight, dễ vận hành, CNCF graduated
CiliumeBPF-based, không cần sidecar, performance cao
Khi nào cần Service Mesh? Chỉ khi có 10+ services với yêu cầu mTLS, advanced traffic management, hoặc fine-grained observability. Đừng thêm complexity nếu 5 services đang communicate tốt qua plain HTTP.

Container Registry

Khái niệm

Container Registry là kho lưu trữ và phân phối container images. Tương tự npm cho Node.js packages hoặc Maven Central cho Java.

Các loại

RegistryLoạiUse case
Docker HubPublic SaaSOpen source images, base images
HarborSelf-hosted (CNCF)Enterprise, vulnerability scanning, signing
ECR/ACR/GCRCloud-managedTích hợp native với cloud provider
OKD Internal RegistryBuilt-inTích hợp sẵn trong OpenShift cluster
GitLab Container RegistryIntegratedCI/CD pipeline integration

Best practices

  • Scan images cho vulnerabilities trước khi push (Trivy, Clair)
  • Sign images (Cosign, Notary) — đảm bảo integrity
  • Tag theo semantic versioning, không chỉ latest
  • Retention policies — xóa images cũ tự động

Monitoring & Observability

3 trụ cột Observability

PillarTrả lờiTools
MetricsHệ thống đang như thế nào? (numbers over time)Prometheus, Mimir, Thanos
LogsChuyện gì đã xảy ra? (events)Loki, ELK Stack, Fluentd

LGTM Stack (phổ biến nhất 2024-2025)

Loki (Logs) + Grafana (Viz) + Tempo (Traces) + Mimir (Metrics)

OpenTelemetry (OTel)

Standard mở cho instrumentation — thu thập metrics, logs, traces từ application rồi gửi đến bất kỳ backend nào (Prometheus, Jaeger, Datadog...). Vendor-neutral, tránh lock-in.

Message Queue / Event Streaming

Khái niệm

Message systems cho phép services giao tiếp asynchronous — producer gửi message, consumer nhận khi sẵn sàng. Decouples services, improves resilience.

2 mô hình

Message QueueEvent Streaming
Message consumed = removedEvents persisted, replay anytime
Point-to-point hoặc pub/subPub/sub với retention
RabbitMQ, ActiveMQApache Kafka, Pulsar, Redpanda
Task distribution, RPCEvent sourcing, data pipelines

Apache Kafka — Event Streaming Platform

  • Distributed commit log — ordered, immutable, durable
  • High throughput — millions messages/second
  • Scalable — partition-based parallelism
  • Durable — replicated across brokers
  • Use cases: event-driven microservices, data pipelines, real-time analytics
Case Study ngân hàng Kafka là backbone cho event-driven banking: mỗi transaction → Kafka event → consumed bởi CRM (cập nhật profile), Loyalty (tính điểm), Fraud Detection (phân tích realtime), Notification (gửi SMS). Decoupled hoàn toàn.

Database trong Cloud Native

Polyglot Persistence

Trong microservices, mỗi service chọn database phù hợp nhất cho workload — không bắt buộc dùng chung 1 loại.

LoạiPhù hợp choVí dụ
Relational (SQL)Transactions, structured data, ACIDPostgreSQL, MySQL, Oracle
Document (NoSQL)Flexible schema, JSON dataMongoDB, CouchDB
Key-ValueCache, session, high-speed lookupRedis, Memcached
Wide-ColumnTime-series, IoT, analyticsCassandra, ScyllaDB
GraphRelationships, social networksNeo4j, Amazon Neptune
SearchFull-text search, logsElasticsearch, OpenSearch

Database trên Kubernetes

  • StatefulSets — đảm bảo stable identity và persistent storage
  • Operators — automate database lifecycle (backup, failover, scaling)
  • Ví dụ: CloudNativePG (PostgreSQL), Strimzi (Kafka), Redis Operator
