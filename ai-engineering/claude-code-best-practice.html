Claude Code Best Practice

From vibe coding to agentic engineering — tổng hợp 83+ best practices từ community

Nguồn: github.com/shanraisshan/claude-code-best-practice — 61.4k stars, community-curated by @shanraisshan

Kho tài liệu trung tâm hệ thống hóa best practices từ community cho Claude Code, giúp developer chuyển từ "vibe coding" sang agentic engineering có cấu trúc. Được hỗ trợ bởi Disrupt.com và ClaudeKit.

Tư duy cốt lõi: Claude Code không chỉ là autocomplete thông minh hơn — nó là lập trình viên AI được trao quyền truy cập hệ thống. Hiệu quả phụ thuộc vào cách bạn quản lý context, cấu trúc workflow, và delegate work cho subagents.

Prompting

Challenge Claude

Yêu cầu Claude "grill me on changes" — không tạo PR cho đến khi bạn pass test của nó. Buộc Claude defend giải pháp thay vì chỉ chấp nhận.

Elegant Solution

Sau các fix tầm thường, yêu cầu Claude bỏ hết và implement giải pháp elegant hơn. Prototype 20–30 phiên bản rẻ thay vì viết spec quá kỹ.

Minimal Input

Paste bug và nói "fix" — không cần micromanage từng bước. Để Claude tự quyết approach.

Planning

  • Luôn bắt đầu session bằng plan mode
  • Dùng AskUserQuestion tool để Claude phỏng vấn bạn thu thập context trước khi code
  • Chia PRD thành vertical slices (tracer bullets) — mỗi slice đi xuyên qua tất cả các layer
  • Prototype 20–30 phiên bản rẻ thay vì viết spec quá kỹ
Vertical Slices: Thay vì build từng layer (DB → API → UI), build một feature end-to-end trước. Mỗi slice chứng minh system hoạt động xuyên suốt. Sau đó mở rộng sang feature tiếp.

Context Management

Context Rot: Chất lượng Claude giảm rõ rệt sau 300–400k tokens. "Dumb zone" xuất hiện ở ~40% context utilization — Claude bắt đầu kém đi từ sớm hơn bạn nghĩ.
  • Giữ context dưới 30–40% cho công việc phức tạp
  • Rewind qua các attempt thất bại thay vì để corrections chồng chất
  • Dùng subagents để cô lập exploration — chỉ trả về kết luận về main context
  • "Summarize from here" trước khi rewind để tạo handoff notes

Session Management

Mỗi turn là một branching point với 5 lựa chọn:

Continue | /rewind | /clear | /compact | Subagent
  • Task mới = session mới
  • Dùng "summarize from here" trước khi rewind để tạo handoff notes
  • Match session boundaries với task boundaries

8 Key Practices

  1. Plan first — Bắt đầu mọi session bằng planning có cấu trúc
  2. Manage context aggressively — Giữ dưới 40%; rewind thay vì correct
  3. Use subagents for delegation — Cô lập exploration, chỉ trả kết luận
  4. Vertical slices over horizontal phases — Build end-to-end trước khi polish từng layer
  5. Prototype proliferation — Build nhiều phiên bản rẻ thay vì over-specify
  6. Challenge Claude — Yêu cầu grill, prove, hoặc scrap và rebuild
  7. Session hygiene — Match session boundaries với task boundaries
  8. Embrace agents — Để autonomous agents xử lý multi-step workflows phức tạp
