Skills for Real Engineers

Bộ agent skills composable của Matt Pocock — thiết kế cho practical software engineering, giải quyết 4 failure modes phổ biến trong AI-assisted coding.

Nguồn: github.com/mattpocock/skills — by Matt Pocock, MIT License, 149k stars, 12.9k forks
Tư duy cốt lõi: Skills không phải framework nặng — đây là các công cụ nhỏ, composable, có thể hack và adapt, hoạt động với bất kỳ model nào. Mục tiêu: xây dựng feedback loops, dùng deliberate small steps, thiết kế có chủ ý, và duy trì shared language giữa người và agent.

Bộ skills được Matt Pocock chia sẻ trực tiếp từ thư mục .claude cá nhân của ông. Được khoảng 60,000 developers đăng ký theo dõi qua newsletter. Thiết kế để giải quyết những vấn đề thực tế nhất khi làm việc với AI coding agents.

4 Failure Modes trong AI-Assisted Development

Failure ModeVấn đềGiải pháp
MisalignmentAgent xây dựng thứ khác với những gì bạn muốn/grill-me, /grill-with-docs — phỏng vấn requirements chi tiết trước khi bắt đầu
VerbosityAgent dùng ngôn ngữ không nhất quán, thiếu project contextCONTEXT.md — thiết lập shared domain language, giảm token spend
Broken CodeOutput không hoạt động vì thiếu feedback loops/tdd, /diagnosing-bugs — automated tests, static typing, red-green-refactor
Poor ArchitectureCodebase trở nên khó maintain và thay đổi/improve-codebase-architecture, /to-prd, /codebase-design — thiết kế có chủ ý

Engineering Skills — Model-Invoked

Các skills này được model tự động gọi trong quá trình làm việc, không cần user trigger trực tiếp.

SkillMô tả
diagnosing-bugsStructured debugging loop theo scientific method: Reproduce → Minimize → Hypothesize → Instrument → Fix root cause (không phải symptom)
tddRed-green-refactor test-driven development: viết test fail trước (Red), viết code tối thiểu để test pass (Green), clean up mà không break tests (Refactor). Feedback loop liên tục đảm bảo code luôn đúng ở mỗi bước

CONTEXT.md — Shared Domain Language

CONTEXT.md là file đặc biệt thiết lập domain-specific terminology cho project. Đây là giải pháp chính cho failure mode Verbosity.

Tác dụng:

  • Giảm token usage — agent không cần giải thích lại terms
  • Tăng code consistency — naming conventions rõ ràng
  • Giảm verbosity — agent biết project dùng từ gì và nghĩa của chúng
  • Improve naming consistency qua toàn bộ codebase
# CONTEXT.md — ví dụ
## Domain Terms
- "Workspace" = a user's isolated environment (not OS workspace)
- "Pipeline" = sequence of transforms on ingest data
- "Shard" = horizontal partition of the events table

## Conventions
- All API responses use camelCase
- Error codes follow: ERR_{DOMAIN}_{SPECIFIC}
- Dates are always UTC ISO-8601
Tip: CONTEXT.md giải quyết failure mode Verbosity. Khi agent biết domain terms, nó viết code đúng naming convention ngay lần đầu — không cần sửa đi sửa lại. Dùng /grill-with-docs để xây dựng và duy trì file này.

Triết lý

Bộ skills này nhấn mạnh rằng các software engineering fundamentals vẫn cực kỳ quan trọng ngay cả khi có AI assistance:

  • Feedback loops: Automated tests, static typing, version control — đây là những gì ngăn code bị broken
  • Deliberate small steps: Tránh big-bang implementations, ưu tiên incremental progress có thể verify
  • Intentional design: Không để complexity tự phát sinh — chủ động thiết kế architecture
  • Shared language: Con người và agent cần dùng cùng terminology để tránh misalignment
