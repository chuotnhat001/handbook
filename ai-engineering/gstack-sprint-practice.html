GStack Sprint Practice

Hướng dẫn thực hành — 23 slash commands biến Claude Code thành đội kỹ sư ảo

Nguồn: github.com/garrytan/gstack — by Garry Tan (Y Combinator CEO), MIT License
Tư duy cốt lõi: Một developer + GStack có thể ship sản phẩm nhanh như team 20 người. Năng suất 810× so với cách truyền thống nhờ 23 vai trò chuyên gia được tự động hóa qua slash commands.

Cài đặt (30 giây)

Yêu cầu: Claude Code, Git, Bun v1.0+ (Node.js trên Windows)

# Clone và setup
git clone https://github.com/garrytan/gstack ~/.claude/skills/gstack
cd ~/.claude/skills/gstack && ./setup

# (Optional) Team mode — auto-update cho shared repos
./setup --team

Gỡ cài đặt:

~/.claude/skills/gstack/bin/gstack-uninstall
# hoặc giữ config:
~/.claude/skills/gstack/bin/gstack-uninstall --keep-state

Sprint Workflow

Think → Plan → Build → Review → Test → Ship → Reflect
  1. Mô tả ý tưởng cho Claude
  2. /office-hours → tạo design doc
  3. /plan-ceo-review → challenge scope
  4. /plan-eng-review → lock architecture
  5. Approve plan, implement
  6. /review → code review + auto-fixes
  7. /qa → test và fix bugs
  8. /ship → merge và deploy
  9. /retro → reflect on sprint

Safety & Security

CommandChức năng
/carefulWarn trước destructive commands (rm -rf, DROP TABLE, force-push)
/freezeRestrict edits to one directory only
/guard/careful + /freeze combined
/csoOWASP Top 10 + STRIDE threat modeling audit
Privacy: Telemetry mặc định OFF. Khi opt-in, chỉ gửi: skill name, duration, success/fail, version, OS. Không bao giờ gửi code, file paths, prompts, hay user content. Đổi bất cứ lúc nào: gstack-config set telemetry off

Parallel Sprints

GStack hỗ trợ 10-15 parallel sprints cùng lúc dùng Conductor. Mỗi sprint là một isolated workspace riêng biệt, cho phép work trên nhiều feature cùng lúc mà không conflict.
