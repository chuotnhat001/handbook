Open Design

Local-first, open-source alternative to Claude Design — ứng dụng desktop native cho macOS và Windows, cho phép AI-powered design workflows không bị vendor lock-in, với 100+ design skills, 150 brand-grade design systems, và 261 plugins.

Nguồn: github.com/nexu-io/open-design — Apache-2.0 License, 72.4K stars, 8.2K forks
Tư duy cốt lõi: Open Design không phải là một agent — nó tích hợp với các coding agent CLI đã có sẵn trên máy của bạn (22+ CLIs), kết hợp 150 brand-grade DESIGN.md systems để tạo ra thiết kế chất lượng chuyên nghiệp mà không bị khóa vào bất kỳ vendor nào.

Open Design là ứng dụng desktop mã nguồn mở tái triển khai Claude Design (closed-source) theo hướng local-first. Toàn bộ hoạt động trên máy người dùng, không telemetry mặc định, hỗ trợ BYOK (Bring Your Own Key) với nhiều provider.

Cài đặt

Desktop App (Khuyến nghị)

Tải từ open-design.ai hoặc GitHub Releases.

  • macOS: Apple Silicon + Intel x64
  • Windows: x64
  • Linux: AppImage (optional)

Zero configuration — tự động phát hiện coding agents đã cài đặt.

Docker Deployment

git clone https://github.com/nexu-io/open-design.git
cd open-design/deploy
cp .env.example .env
echo "OD_API_TOKEN=$(openssl rand -hex 32)" >> .env
docker compose up -d
# Truy cập: http://localhost:7456

Build từ Source

git clone https://github.com/nexu-io/open-design.git
cd open-design
corepack enable && pnpm install
pnpm tools-dev run web

Yêu cầu: Node ~24, pnpm 10.33.x

CLI Integration (không cần UI)

od mcp install claude          # hoặc: codex, cursor, copilot, ...
# Sau đó dùng trong agent: "Use open-design to generate a landing page..."

Workflow

Quy trình từ brief đến artifact hoàn chỉnh:

Brief→ Plugin→ Direction→ Design System→ Artifact→ Handoff
  1. Submit design brief; plugin picker cung cấp templates (landing page, deck, dashboard, v.v.)
  2. Lock creative direction (curated options hoặc import từ screenshot/URL)
  3. Agent sinh artifact đầu tiên, stream vào sandboxed iframe (có thể chỉnh sửa trực tiếp)
  4. Handoff sang engineering: artifact là real HTML/CSS dùng được cho Cursor/Claude Code
  5. Export sang PPTX/PDF/MP4 cho marketing hoặc stakeholders

Multi-Artifact Generation

Open Design hỗ trợ tạo nhiều loại artifact khác nhau:

  • Web, desktop, mobile prototypes — single-page HTML
  • Live dashboards và decision rooms
  • Presentation decks — 15+ templates, 36+ themes
  • AI-generated images — editorial, cinematic, product
  • HyperFrames — HTML → MP4 motion graphics (tích hợp HeyGen's OSS framework)

Export formats: HTML (inlined), PDF (print), PPTX (agent-driven), ZIP, Markdown, MP4 (HyperFrames)

Design Systems & Branding

Bao gồm 150 pre-built DESIGN.md systems, mỗi system gồm 9 sections:

  • Color palettes, typography, spacing, layout
  • Components, motion, voice
  • Brand guidelines, anti-patterns

Các brand được đại diện: Linear, Stripe, Vercel, Airbnb, Apple, Tesla, Notion, Figma, Cursor, Supabase, và 140+ thương hiệu khác.

Lợi ích chính: DESIGN.md là plain text, version-controllable, đọc được bởi bất kỳ coding agent nào — không phụ thuộc vào proprietary format.

Plugin & Skill System

261 official plugins được tổ chức theo task:

  • Scenarios, image templates, video templates
  • Design systems, UI atoms, examples

100+ reusable skills: saas-landing, dashboard, mobile-app, pm-spec, email-marketing, v.v.

Plugin registry với metadata, inputs, pipelines, và capability declarations. Hỗ trợ community plugins bên cạnh official bundled plugins.

Technical Architecture

┌─ Browser (Next.js 16) / Electron Shell ─┐
│   Chat · Workspace · Preview · Settings │
└─────────────────┬───────────────────────┘
                  │ /api/*
                  ▼
    ┌──────────────────────────────┐
    │  Local Daemon (Node/Express) │
    │  SQLite backend              │
    │  /api/skills, /plugins, etc. │
    │  MCP stdio server            │
    └──────────┬───────────────────┘
               │ spawn(cli)
               ▼
    ┌──────────────────────────────┐
    │  Coding Agent (22+ CLIs)     │
    │  Reads SKILL.md + DESIGN.md  │
    │  Writes artifacts to disk    │
    └──────────────────────────────┘

Daemon chạy loopback-only (127.0.0.1) theo mặc định. LAN exposure yêu cầu thiết lập explicit OD_BIND_HOST + OD_ALLOWED_ORIGINS.

Privacy & Security

  • Local-first: Mọi thứ chạy trên máy người dùng — không telemetry, không cloud round-trip theo mặc định
  • BYOK proxy: Khi cần network, SSRF-guarded tại daemon edge; block internal IPs, link-local, CGNAT
  • Daemon binding: Loopback-only mặc định (127.0.0.1); LAN exposure yêu cầu config tường minh
  • MCP read-only: Skills và design systems accessible cho agents qua structured API; connector credentials ở loopback-only

Plugin Development

Cấu trúc plugin tối thiểu:

my-plugin/
├── SKILL.md             # Bắt buộc: YAML frontmatter + workflow
├── open-design.json     # Marketplace metadata (name, version, inputs, capabilities)
├── README.md            # Usage guide
├── preview/             # index.html hoặc poster.png
└── examples/            # Use cases

Scaffold và validate:

od plugin scaffold --id my-plugin --title "My Plugin"
od plugin validate ./my-plugin

Các field quan trọng trong open-design.json:

  • specVersion: 1.0.0
  • name: stable ID
  • od.kind: skill / scenario / atom / bundle
  • od.mode: prototype / deck / image / video
  • od.capabilities[], od.inputs[]

Đường dẫn đóng góp: plugins/community/ cho third-party; plugins/_official/ cho bundled.
