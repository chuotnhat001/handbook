Skills Stack

Mô hình 3 lớp: Decision / Context / Execution — kết hợp GStack, GSD Core và Superpowers

Nguồn: Superpowers + GStack + GSD Core — Three-layer skills architecture
Tư duy cốt lõi: Ba tool giải quyết ba câu hỏi khác nhau theo thứ tự: GStack hỏi "Nên build gì?", GSD hỏi "Đang build đến đâu?", Superpowers hỏi "Build cụ thể ra sao?". Chạy sai thứ tự hoặc bỏ lớp giữa là nguồn gốc của context rot và scope creep.

Layer 1 — Decision: GStack

Repo: garrytan/gstack — MIT License

GStack cung cấp 23 vai trò chuyên biệt (CEO, Designer, Engineering Manager, QA Lead, Security Officer, Release Engineer...) hoạt động qua slash commands, giúp đưa ra quyết định sản phẩm và kỹ thuật trước khi viết code.

CommandVai trò
/office-hoursPhỏng vấn kiểu YC, stress-test ý tưởng bằng câu hỏi buộc phải đối mặt
/plan-ceo-reviewCEO review scope và ambition, thách thức giả định
/plan-eng-reviewLock architecture, data flow, edge cases trước khi build
/plan-design-reviewAudit design với taste calibration, phát hiện AI slop
/reviewStaff engineer audit, phát hiện bug production thoát qua CI
/csoOWASP + STRIDE security audit
/shipTự động hoá PR + deploy

Vai trò trong mô hình: Trả lời câu hỏi "Có nên build không? Build cái gì? Build như thế nào về mặt kiến trúc và sản phẩm?" trước khi chạm vào code.

Layer 2 — Context: GSD Core

Repo: open-gsd/gsd-core — Install: npx @opengsd/gsd-core@latest

Framework meta-prompting và context engineering, giúp AI coding agent duy trì trạng thái và context qua nhiều session dài mà không bị "context rot".

FeatureMô tả
Phase Loop 5 bướcDiscuss → Plan → Execute → Verify → Ship, lặp theo milestone
Subagent delegationResearch/execution nặng chạy trong context window 200k sạch riêng
Persistent artifactsSTATE.md, CONTEXT.md, PLAN.md — memory cross-session
Multi-runtimeClaude Code, Gemini CLI, Copilot, Cursor, Windsurf

Vai trò trong mô hình: Trả lời câu hỏi "Chúng ta đang ở đâu? Task tiếp theo là gì? Quyết định đã lock là gì?" Là lớp trung gian giữ memory và điều phối execution theo kế hoạch.

Layer 3 — Execution: Superpowers

Repo: obra/Superpowers — Cross-platform (Claude Code, Cursor, GitHub Copilot, Gemini, Kimi...)

Methodology toàn diện cho coding agent, tập trung vào thực thi code chất lượng cao theo chu trình TDD, với subagent coordination và systematic debugging.

SkillMô tả
test-driven-developmentRED-GREEN-REFACTOR cycle bắt buộc, không exception
writing-plansBreak work thành task 2-5 phút với exact specification
executing-plansDispatch subagent xử lý song song
using-git-worktreesIsolated workspace cho parallel dev
systematic-debuggingRoot cause analysis, không patch triệu chứng
requesting-code-reviewSeverity-based review (CRITICAL/HIGH/MEDIUM/LOW)
brainstormingRefine ý tưởng qua câu hỏi, khám phá alternatives

Vai trò trong mô hình: "Build như thế nào từng dòng code?" Đảm bảo mỗi task đi qua chu trình kiểm chứng trước khi claim done.

Workflow kết hợp đầy đủ

PHASE 1 — DECISION (GStack)
  /office-hours          ← stress-test ý tưởng
  /plan-ceo-review       ← validate scope & ambition
  /plan-eng-review       ← lock architecture, data flow
  [/plan-design-review]  ← chỉ khi có UI
  ↓ Gate 1: scope approved, architecture locked

PHASE 2 — CONTEXT (GSD Core)
  /gsd-new-project       ← init STATE.md, ROADMAP.md
  /gsd-discuss-phase     ← gather context cho milestone
  /gsd-plan-phase        ← tạo PLAN.md với atomic tasks
  ↓ Gate 2: artifacts khớp Gate 1, tasks fit 1 context window

PHASE 3 — EXECUTION (Superpowers)
  using-git-worktrees    ← isolate workspace
  writing-plans          ← break tasks 2-5 min (skip nếu PLAN.md đủ chi tiết)
  executing-plans        ← dispatch subagents song song
  test-driven-development← RED → GREEN → REFACTOR
  systematic-debugging   ← khi gặp lỗi
  ↓ Gate 3: mọi task pass TDD cycle

PHASE 4 — REVIEW (GStack + Superpowers)
  requesting-code-review ← review theo plan, severity-based
  /review                ← staff audit catch production bug
  [/cso]                 ← security audit khi cần
  [/qa]                  ← live browser test khi có UI
  ↓ Gate 4: không còn CRITICAL issue

PHASE 5 — SHIP (GStack + GSD)
  finishing-a-development-branch
  /ship                  ← sync main, push, open PR
  /land-and-deploy       ← merge, CI, deploy verify
  Update STATE.md / DECISIONS.md

Gates & Checkpoints

GateĐiều kiệnNếu fail
Gate 1Scope approved, architecture lockedQuay lại Decision layer
Gate 2Artifacts khớp Gate 1, tasks fit context windowChia nhỏ hoặc revisit scope
Gate 3Mọi task pass TDD cycleFix tại Execution layer
Gate 4Không CRITICAL issue, tests passMinor bug → Gate 3; Architectural issue → Gate 1

Scale theo task size

Task typeLayers sử dụng
2-line fix, config changeSuperpowers light mode only. Skip Phases 1, 2
Prototype, single-session projectGStack (decision) + Superpowers (execution). Skip GSD
Large feature, long project, multi-streamFull 5 phases + parallel subagents

Anti-patterns cần tránh

Lãng phí token: Chạy full ceremony (cả 3 lớp) cho task 2-line fix — tốn 10k+ token vô nghĩa.
  • Bỏ qua GSD với project dài → context rot sau vài session
  • Chạy cả /gsd-plan-phase lẫn writing-plans cho cùng scope → double planning
  • Nhảy thẳng vào Superpowers khi scope chưa lock → scope creep mid-execution
  • Claim "done" mà không qua Gate 3 (TDD pass) hoặc Gate 4 (review clean)
  • Bật tất cả GStack review roles cho task nhỏ → burns 10k+ tokens không cần thiết
