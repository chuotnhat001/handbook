⚡ Superpowers

Agentic Skills Framework — composable skills định hướng coding agents qua structured workflows, autonomous nhưng có kiểm soát

Nguồn: github.com/obra/Superpowers — by Jesse Vincent / Prime Radiant, MIT License

Superpowers là một software development methodology được thiết kế dành riêng cho coding agents. Thay vì để agents tự do thực thi ngay lập tức, Superpowers hướng dẫn agents đi qua các bước có cấu trúc: từ clarify requirements, present designs để validate theo từng phần nhỏ, lập kế hoạch chi tiết, rồi mới execute với các review cycles nghiêm ngặt.

Project được duy trì bởi Jesse Vincent và Prime Radiant, với community support qua Discord, GitHub Issues và release announcements tại primeradiant.com.

Tư duy cốt lõi: Agents không nên tự quyết định và code ngay — hãy đưa chúng đi qua deliberate process: clarify, design, plan, implement, verify. "Verify before declaring success" là nguyên tắc bất biến.

Core Workflow — 6 giai đoạn

Brainstorm→ Git Isolate→ Plan→ Execute→ TDD→ Review & Merge

Giai đoạn 1: Brainstorming

Trước khi viết bất kỳ dòng code nào, agent phải đi qua bước clarify requirements:

  • Refine ideas thông qua câu hỏi có mục đích
  • Khám phá các alternatives — không chốt ngay giải pháp đầu tiên
  • Validate designs theo từng phần nhỏ, tiêu hóa được (digestible sections)
  • Chỉ tiến lên bước tiếp theo khi design đã được phê duyệt
Lưu ý: Brainstorming không phải bước tùy chọn. Đây là gate đảm bảo agent hiểu đúng yêu cầu trước khi đầu tư effort vào implementation.

Giai đoạn 2 & 3: Git Isolation + Writing Plans

Git Isolation

  • Tạo isolated workspace trên new branch trước khi bắt đầu
  • Verify clean test baseline — tất cả tests phải pass trước khi thêm code mới
  • Đảm bảo môi trường clean, không bị ảnh hưởng bởi work-in-progress khác

Writing Plans

Kế hoạch tốt phải đáp ứng các tiêu chí sau:

  • 2-5 minute tasks — mỗi task đủ nhỏ để hoàn thành nhanh và verify ngay
  • Exact file paths — không mơ hồ về nơi thực hiện thay đổi
  • Verification steps — mỗi task có cách kiểm tra kết quả cụ thể
  • Xác định dependencies giữa các tasks
  • Lên kế hoạch cho edge cases và error scenarios

Giai đoạn 4: Execution

Superpowers cung cấp hai mode execution:

ModeKhi nào dùngCơ chế
Subagent-Driven DevelopmentTasks phức tạp, cần isolation caoDispatch fresh agent cho mỗi task; two-stage review: (1) spec compliance, (2) code quality
Parallel AgentsNhiều tasks độc lập cùng lúcChạy song song để giảm tổng thời gian; merge kết quả sau khi tất cả pass
Subagent two-stage review: Stage 1 kiểm tra agent có làm đúng spec không. Stage 2 đánh giá chất lượng code. Hai giai đoạn tách biệt giúp phát hiện vấn đề sớm hơn.

Giai đoạn 5: Test-Driven Development

TDD trong Superpowers không phải tùy chọn — đây là requirement bắt buộc:

RED (Fail)→ GREEN (Pass)→ REFACTOR
  1. RED — Viết failing test trước. Nếu có pre-existing code, xóa đi và bắt đầu lại từ test
  2. GREEN — Viết implementation tối thiểu để test pass — không hơn không kém
  3. REFACTOR — Cải thiện code quality trong khi giữ nguyên tests xanh
Anti-pattern: Không được viết implementation rồi mới viết test để "cover" code đã có. Thứ tự phải là: test trước, implementation sau. Đây là điểm phân biệt TDD thật sự với test-after.

Giai đoạn 5: Code Review

Code review trong Superpowers được thực hiện đối chiếu trực tiếp với plan:

  • Review từng thay đổi so với task trong plan đã phê duyệt
  • Severity-based issue reporting — phân loại mức độ nghiêm trọng của từng vấn đề
  • Không merge cho đến khi tất cả critical issues được giải quyết

Giai đoạn 6: Branch Finishing

Khi implementation hoàn tất, agent trình bày các lựa chọn rõ ràng:

Lựa chọnKhi nào phù hợp
MergeFeature hoàn chỉnh, tests pass, review approved
Pull RequestCần human review trước khi merge vào main
Keep (as-is)Work-in-progress, chưa sẵn sàng merge
DiscardApproach sai, cần bắt đầu lại

Branch cleanup được thực hiện sau khi quyết định được đưa ra.
