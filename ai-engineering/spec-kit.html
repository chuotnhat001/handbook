Spec Kit

Toolkit mã nguồn mở của GitHub cho Spec-Driven Development (SDD) — biến specifications thành executable artifacts, trực tiếp generate working implementations thay vì chỉ mô tả chúng

Nguồn: github.com/github/spec-kit — by GitHub, MIT License

Spec Kit là toolkit mã nguồn mở của GitHub cho Spec-Driven Development (SDD) — một methodology đảo ngược quy trình phát triển truyền thống. Thay vì viết code từ đầu, developers định nghĩa specifications trước, và các specifications đó trực tiếp generate working implementations. Toolkit hỗ trợ hơn 30 AI coding agents và hoạt động với GitHub Copilot, Claude Code, Gemini CLI, Cursor, và nhiều công cụ khác.

Tư duy cốt lõi: Specifications trở thành executable artifacts — không chỉ guide implementation mà trực tiếp generate nó. Phương pháp này đặt WHAT và WHY trước HOW, giúp teams tập trung vào product scenarios và predictable outcomes thay vì "vibe coding" từng phần từ đầu.

Triết lý Spec-Driven Development

"Specifications become executable, directly generating working implementations rather than just guiding them."

SDD đảo ngược trình tự phát triển truyền thống bằng cách đặt intent-driven workflows lên trước — tức là xác định cái gì cần làm và tại sao trước khi quyết định làm thế nào.

Truyền thống vs Spec-Driven

Traditional DevelopmentSpec-Driven Development (SDD)
Specs chỉ mô tả → developer implementSpecs executable → trực tiếp generate implementation
HOW trước (chọn tech stack → build)WHAT và WHY trước (requirements → tech → build)
Specs trở nên outdated sau khi code thay đổiSpecs là source of truth, luôn sync với implementation
Gắn chặt với tech stack cụ thểTechnology independent — cùng spec, nhiều stacks khác nhau
"Vibe coding" từng phần từ đầuFocus vào product scenarios và predictable outcomes

Nguyên tắc cốt lõi

  • Specifications are executable — Không chỉ là documentation, mà là input trực tiếp cho generation
  • Intent over implementation — Focus vào user needs và requirements, không phải technical details
  • Technology independence — Specs không bị tied to bất kỳ framework hay language nào
  • Iterative refinement — Chu kỳ Clarify → Plan → Validate → Execute lặp lại cho đến khi đủ chính xác

Development Phases

Spec Kit hỗ trợ ba kịch bản phát triển chính:

PhaseMô tảUse case
0-to-1 (Greenfield)Build application từ scratch dựa trên specificationsNew projects, MVPs, prototypes
Creative ExplorationParallel implementations trên nhiều tech stacks khác nhau từ cùng một specPrototyping, tech stack evaluation
Iterative Enhancement (Brownfield)Thêm features và modernize existing systemsLegacy codebases, feature additions

7-Step Workflow

1. Constitution→ 2. Specification→ 3. Clarification→ 4. Plan→ 5. Task Breakdown→ 6. Validation→ 7. Implementation
  1. Constitution — Thiết lập project principles, development guidelines, và governance rules cho toàn bộ project
  2. Specification — Mô tả requirements, tập trung vào functional needs thay vì technology choices
  3. Clarification — Giải quyết các underspecified areas và ambiguities trước khi bước vào planning
  4. Plan — Xác định tech stack và architecture choices phù hợp với specs đã được làm rõ
  5. Task Breakdown — Generate actionable, ordered tasks từ technical plan
  6. Validation — Audit plan cho completeness và alignment (cross-artifact consistency check)
  7. Implementation — Execute tasks một cách có hệ thống dựa trên tất cả artifacts đã tạo
Không phải one-shot: Workflow có tính iterative. Sau mỗi bước, có thể quay lại clarify hoặc re-specify. /speckit.analyze có thể validate cross-artifact consistency ở bất kỳ thời điểm nào trong quá trình, không chỉ ở bước 6.

Core Commands

Spec Kit cung cấp các slash commands để dùng trực tiếp với AI coding agents:

CommandChức năng
/speckit.constitutionThiết lập project principles và governance
/speckit.specifyTạo functional specifications (tập trung vào requirements, không phải tech)
/speckit.planXây dựng technical implementation strategies
/speckit.tasksGenerate actionable task breakdowns theo thứ tự
/speckit.implementExecute full implementation plan một cách có hệ thống
/speckit.clarifyResolve specification ambiguities và gaps
/speckit.analyzeValidate cross-artifact consistency
/speckit.checklistGenerate quality validation checklists

Optional Commands

CommandChức năng
/speckit.taskstoissuesConvert tasks thành GitHub issues tự động
Lưu ý về platform: Tùy vào AI agent được sử dụng, commands có thể dùng dưới dạng slash commands hoặc skills mode. Xem tài liệu của từng integration để biết cú pháp phù hợp.

Customization Architecture

Spec Kit hỗ trợ ba lớp customization, resolve theo thứ tự ưu tiên từ cao xuống thấp:

LoạiMục đíchVí dụ
ExtensionsThêm capabilities mới vào toolkitDomain-specific workflows, external integrations
PresetsCustomize existing workflows và terminologyCompliance formats, organizational standards
BundlesPackage curated sets of extensions và presetsRole-based provisioning (backend, frontend, data)
Project-local overridesOverride templates cho project cụ thể.specify/templates/overrides/

Resolution order (cao đến thấp): Project overrides → Presets → Extensions → Core templates

  • Templates resolve tại runtime, cho phép flexible customization không cần rebuild
  • Bundle operations là idempotent và chỉ ảnh hưởng trong phạm vi project scope
  • Community có thể publish custom extensions và presets lên marketplace

Project Structure

Sau khi khởi tạo với specify init, project sẽ có cấu trúc sau:

my-project/
├── .specify/                    ← Cấu hình và customization
│   ├── memory/
│   │   └── constitution.md      ← Project principles và governance
│   ├── templates/               ← Template overrides cho project này
│   └── scripts/                 ← Custom automation scripts
├── specs/                       ← Feature specifications
│   └── [feature-name]/
│       ├── spec.md              ← Functional requirements
│       ├── plan.md              ← Technical plan và architecture
│       ├── tasks.md             ← Task breakdown theo thứ tự
│       └── (supporting artifacts)
├── contracts/                   ← API contracts
├── data-models/                 ← Data schemas
└── research/                    ← Research documents

Key Guarantees

  • Multi-step refinement — Giảm rework so với single-shot generation bằng cách làm rõ requirements trước khi implement
  • Runtime template resolution — Templates resolve tại runtime, cho phép customization linh hoạt
  • Idempotent bundle operations — Bundle operations confined to project scope, không ảnh hưởng hệ thống
  • Offline support — Tất cả consume/author commands hoạt động được khi offline

Installation

Requirements

  • Linux / macOS / Windows
  • Python 3.11+
  • Git
  • uv hoặc pipx package manager
  • Một AI coding agent được hỗ trợ
# Cài đặt specify CLI (thay vX.Y.Z bằng phiên bản cụ thể)
uv tool install specify-cli --from git+https://github.com/github/spec-kit.git@vX.Y.Z

# Khởi tạo project mới với GitHub Copilot
specify init my-project --integration copilot

# Khởi tạo project với Claude Code
specify init my-project --integration claude-code
Phiên bản: Thay vX.Y.Z bằng tag phiên bản cụ thể từ trang Releases trên GitHub repo. Không dùng @main cho production để tránh breaking changes từ commits chưa stable.
