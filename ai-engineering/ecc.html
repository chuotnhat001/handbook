ECC — The Harness-Native Operator System

Hệ thống operator cho agentic work — tối ưu hiệu suất AI coding agents trên nhiều harnesses

Nguồn: github.com/affaan-m/ECC — MIT License, v2.0.0-rc.1 (April 2026) | 182K+ stars, 170+ contributors

ECC (viết tắt của Enhanced Claude Code) là hệ thống operator toàn diện cho AI coding agents. Thay vì dùng AI agent với prompt mặc định, ECC cung cấp một lớp trung gian giữa developer và AI — bao gồm 63+ agents chuyên biệt, 249+ skills, 29+ bộ rules cho 20+ ngôn ngữ/framework, và hệ thống hooks tự động — giúp agent hoạt động chính xác, hiệu quả, và an toàn hơn đáng kể.

Tư duy cốt lõi: AI agent mạnh nhất khi được cung cấp context đúng vào đúng thời điểm. ECC là lớp "operator system" nằm giữa developer và AI harness — tự động cung cấp agents, skills, rules, và hooks phù hợp cho từng loại task, từng ngôn ngữ, từng giai đoạn của workflow.

ECC được xây dựng từ quy trình kỹ thuật thực tế trên nhiều harnesses, không phải lý thuyết. Mọi component đều đã được kiểm chứng trong production với 997+ bài test nội bộ và hệ thống CI/CD toàn diện.

Những gì ECC giải quyết

  • Phân mảnh context — Agent không biết best practices của ngôn ngữ/framework đang dùng → ECC tự động load rules phù hợp
  • Quá tải tác vụ — Main agent làm tất cả mọi thứ → ECC ủy quyền cho agents chuyên biệt
  • Mất kiến thức — Patterns hay bị quên giữa các sessions → Instinct Learning lưu và tái sử dụng
  • Lỗ hổng bảo mật — Code được sinh ra mà không qua review → AgentShield quét tự động với 102 rules
  • Lãng phí token — Load toàn bộ context không cần thiết → Chỉ load chiến lược những gì cần

Agents (63+)

ECC sử dụng mô hình ủy quyền (delegation model): thay vì main agent xử lý mọi thứ, nó ủy quyền task cho subagents chuyên biệt. Mỗi agent có context, chuyên môn, và bộ công cụ riêng — tối ưu cho loại task được giao.

Mô hình ủy quyền: Main agent nhận task → xác định loại task → ủy quyền cho agent phù hợp → agent chuyên biệt thực thi với context tối ưu. Review code → code-review agent. Quét bảo mật → security agent. Lỗi build → language-specific resolver.

Agent Format

Mỗi agent là file Markdown với YAML frontmatter:

---
name: python-reviewer
description: Reviews Python code for idioms, type safety, and PEP compliance
tools: [Read, Grep, Glob]
model: sonnet
---

## Instructions
Review the provided Python code for...

## Checklist
- [ ] Type hints on all public functions
- [ ] No bare except clauses
...

Delegation Flow Example

# Người dùng yêu cầu review Python code
User: "Review this Python file for issues"

# Luồng ủy quyền ECC:
1. Main agent nhận yêu cầu
2. Phát hiện: Python file → ủy quyền cho python-reviewer agent
3. python-reviewer agent tải:
   - Rules riêng cho Python (PEP, type hints, idioms)
   - Checklist review Python
   - Patterns riêng của project (nếu có)
4. Agent thực thi review với đầy đủ context
5. Trả kết quả cho main agent → người dùng

# Nếu phát hiện vấn đề bảo mật trong review:
6. Chuyển tiếp cho security-reviewer agent
7. Security agent chạy phân tích sâu hơn
8. Báo cáo tổng hợp → người dùng
Chọn model: Mỗi agent có thể chỉ định model trong frontmatter. Tác vụ đơn giản (tra cứu docs, formatting) dùng model nhẹ hơn (sonnet/haiku). Tác vụ phức tạp (review kiến trúc, phân tích bảo mật) dùng model mạnh hơn (opus). Điều này tối ưu cả chi phí lẫn chất lượng.

Rules (29+)

Rules là quy tắc bắt buộc tuân theo — được tải tự động và áp dụng liên tục trong suốt session. Khác với skills (workflow theo yêu cầu), rules luôn active và áp đặt tiêu chuẩn cho mọi code được sinh ra.

Hệ thống kế thừa (Inheritance)

Rules được tổ chức theo hai cấp với cơ chế kế thừa:

rules/
├── common/              ← Lớp nền tảng (chung cho mọi ngôn ngữ)
│   ├── coding-style.md  ← Đặt tên, định dạng, cấu trúc
│   ├── git-workflow.md   ← Phân nhánh, commits, PRs
│   ├── testing.md        ← Chiến lược test, coverage
│   ├── performance.md    ← Nguyên tắc tối ưu
│   ├── patterns.md       ← Mẫu thiết kế, anti-patterns
│   ├── security.md       ← Nguyên tắc bảo mật, OWASP
│   ├── hooks.md          ← Hướng dẫn viết hooks
│   └── agents.md         ← Tiêu chuẩn phát triển agents
│
└── [ngôn ngữ]/          ← Lớp mở rộng (ghi đè common)
    ├── typescript/
    ├── python/
    ├── golang/
    ...
    └── web/
Nguyên tắc ghi đè: Mỗi file riêng theo ngôn ngữ mở đầu bằng "This file extends [common/xxx.md] with [Language] specific content." Khi có xung đột giữa common và language-specific, rules riêng theo ngôn ngữ luôn được ưu tiên — vì idioms của ngôn ngữ quan trọng hơn nguyên tắc chung.

Cách Rules được tải

  1. Bắt đầu session — Common rules luôn được tải
  2. Phát hiện project — Nhận diện ngôn ngữ/framework từ config files (package.json, Cargo.toml, go.mod...)
  3. Tải rules ngôn ngữ — Tải rules riêng theo ngôn ngữ tương ứng
  4. Ghi đè cấp file — Khi chỉnh sửa file cụ thể, rules của ngôn ngữ đó được ưu tiên cao nhất

AgentShield & Security

AgentShield là hệ thống quét bảo mật tích hợp của ECC — chạy tự động, không cần gọi thủ công. Với 102 quy tắc phân tích tĩnh và 1282 bài test, nó quét mọi thành phần của project trước khi code được commit.

Bảo mật trước tiên: AgentShield không chỉ quét code được sinh ra — nó quét cả CLAUDE.md, cấu hình MCP, hooks, và định nghĩa agent. Mọi bề mặt tấn công đều được bao phủ, từ prompt injection đến lỗ hổng dependencies.

102 Static Analysis Rules

Tổ chức theo danh mục:

  • Phòng chống injection — SQL injection, command injection, XSS, LDAP injection, template injection
  • Xác thực & Phân quyền — Mẫu auth yếu, thiếu kiểm soát truy cập, lỗi quản lý session
  • Mật mã học — Thuật toán không an toàn, khóa hardcode, sinh số ngẫu nhiên yếu
  • Rò rỉ dữ liệu — Secrets trong code, rò rỉ PII, thông báo lỗi chi tiết, debug mode trong production
  • Xác thực đầu vào — Thiếu sanitization, path traversal, deserialization không an toàn
  • Cấu hình — Sai cấu hình CORS, thiếu security headers, thông tin mặc định
  • Bảo mật dependency — CVE đã biết, packages không được duy trì, phát hiện typosquatting
  • Riêng cho Agent — Prompt injection, leo thang quyền tool, kết nối MCP không an toàn

Continuous Learning — Instinct v2

Instinct Learning là hệ thống học liên tục của ECC — tự động trích xuất patterns từ sessions thành skills có thể tái sử dụng. Càng dùng nhiều, hệ thống càng hiểu project và workflow của developer, giảm lặp lại và tăng tính nhất quán theo thời gian.

Học tích lũy: Mỗi session là cơ hội học. Patterns thành công được trích xuất, xác thực, và trở thành skills mới. Patterns thất bại được ghi nhận để tránh lặp lại. Kiến thức tích lũy dồn theo thời gian.

Cách hoạt động

# Vòng đời Instinct Learning:

1. QUAN SÁT SESSION
   - Theo dõi workflow patterns trong session hiện tại
   - Ghi nhận quyết định, cách tiếp cận, và kết quả

2. TRÍCH XUẤT PATTERN (Stop hook)
   - Cuối session, đánh giá patterns đáng trích xuất
   - Nhận diện: workflows lặp lại, bản sửa thành công, idioms riêng của project

3. HÌNH THÀNH BẢN NĂNG
   - Chuyển đổi patterns thô thành instincts có cấu trúc
   - Mỗi instinct = điều kiện kích hoạt + hành động khuyến nghị + điểm tin cậy

4. XÁC THỰC
   - Instincts mới được xác thực trước khi kích hoạt
   - Đối chiếu với skills hiện có để tránh xung đột
   - Ngưỡng tin cậy phải đạt trước khi được thăng hạng

5. THĂNG HẠNG THÀNH SKILL
   - Instincts có độ tin cậy cao trở thành skills đầy đủ
   - Skills mới được thêm vào registry
   - Sẵn sàng cho tất cả sessions tương lai

Token Optimization & Model Routing

ECC tối ưu chi phí và hiệu suất thông qua nhiều chiến lược — từ tinh giản system prompt đến lựa chọn model thông minh. Mục tiêu: đạt chất lượng output tương đương hoặc tốt hơn với ít tokens hơn.

Model Routing

Không phải mọi tác vụ đều cần model mạnh nhất. ECC hướng dẫn chọn model phù hợp:

Loại tác vụModel khuyến nghịLý do
Review kiến trúc, suy luận phức tạpOpusCần suy nghĩ sâu, phân tích nhiều bước
Sinh code, tái cấu trúcSonnetCân bằng giữa chất lượng và tốc độ/chi phí
Tra cứu tài liệu, formatHaikuTác vụ đơn giản, phản hồi nhanh, chi phí thấp
Phân tích bảo mậtOpusQuan trọng — cần độ chính xác cao, ít bỏ sót
Sửa lỗi buildSonnetĐối chiếu mẫu từ thông báo lỗi, không cần suy luận sâu
Định tuyến ở cấp agent: Mỗi agent có thể chỉ định model trong YAML frontmatter. Khi ủy quyền tác vụ, agent phù hợp tự động dùng model phù hợp — không cần developer chọn thủ công.

System Prompt Slimming

Giảm lượng token overhead từ system prompts:

  • Tải có điều kiện — Chỉ tải rules/skills liên quan cho context hiện tại, không dump toàn bộ
  • Nén ngôn ngữ — Rules được viết súc tích, tránh giải thích dài dòng trong system prompts
  • Context phân lớp — Rules nền tảng luôn hiện diện, rules theo domain tải khi cần
  • Giới hạn context — ECC_SESSION_START_MAX_CHARS=8000 giới hạn context tải khi bắt đầu session

Installer Technical Details

File install.sh là wrapper bash (32 dòng):

  • Giải quyết symlink — Tìm thư mục gốc thực của repo khi được gọi qua npm bin symlink
  • Tự cài dependency — Tự chạy npm install nếu thiếu node_modules
  • Xử lý đường dẫn đa nền tảng — Phát hiện MSYS2/Git Bash, chuyển đổi đường dẫn POSIX sang Windows
  • Ủy quyền — Gọi scripts/install-apply.js (Node.js runtime) cho logic cài đặt thực tế

Người dùng Windows dùng install.ps1 (tương đương PowerShell).

Bảng điều khiển Desktop

ECC cung cấp bảng điều khiển GUI (ecc_dashboard.py, dựa trên Tkinter):

  • Trình duyệt thành phần — Xem tất cả agents, skills, rules theo danh mục
  • Chuyển đổi giao diện sáng/tối
  • Tổng quan trạng thái — Hooks đang hoạt động, skills đã tải, trạng thái session
  • Thao tác nhanh — Bật/tắt thành phần, chuyển đổi profiles
# Khởi chạy bảng điều khiển
python ecc_dashboard.py

ECC Pro (Optional)

Phiên bản Pro là GitHub App hỗ trợ private repositories. Core ECC hoàn toàn miễn phí theo giấy phép MIT cho mọi mục đích sử dụng bao gồm thương mại. Pro chỉ thêm:

  • Tích hợp private repo
  • Đồng bộ instincts chia sẻ trong nhóm
  • Hỗ trợ ưu tiên
