gstack

AI-Powered Software Factory — biến Claude Code thành một đội engineering ảo với 23 skills chuyên biệt, tổ chức quanh sprint workflow: Think → Plan → Build → Review → Test → Ship → Reflect.

Nguồn: github.com/garrytan/gstack — by Garry Tan (Y Combinator President & CEO), MIT License, 118K stars
Tư duy cốt lõi: gstack là một quy trình, không phải tập hợp công cụ. Mỗi skill trong sprint đọc output của skill trước và nuôi dưỡng skill tiếp theo — /office-hours tạo design doc, /plan-eng-review đọc nó để viết test plan, /qa tự động pick up test plan đó. Không có thông tin nào bị mất giữa các handoff.

gstack là framework mã nguồn mở biến Claude Code thành một đội engineering ảo có phối hợp. Được xây dựng và duy trì bởi Garry Tan (Y Combinator President & CEO), nó cho phép developer cá nhân ship với tốc độ chưa từng có thông qua structured workflows, specialized AI roles, và integrated development tools.

Cài đặt

Yêu cầu

  • Claude Code
  • Git
  • Bun v1.0+
  • Node.js (chỉ Windows)

Quick Setup (30 giây)

git clone --single-branch --depth 1 https://github.com/garrytan/gstack.git \
  ~/.claude/skills/gstack && cd ~/.claude/skills/gstack && ./setup

Team Mode (auto-update cho shared repos)

(cd ~/.claude/skills/gstack && ./setup --team) && \
  ~/.claude/skills/gstack/bin/gstack-team-init required && \
  git add .claude/ CLAUDE.md && git commit -m "require gstack for AI-assisted work"

Team mode bật auto-update mỗi session (throttled 1/giờ, network-failure-safe, silent). Có thể chọn required (block teammate không có gstack) hoặc optional (nudge không block).

Sprint Workflow

gstack cấu trúc phát triển thành pipeline 7 giai đoạn. Mỗi giai đoạn có specialized skills đọc context từ giai đoạn trước và truyền output xuống giai đoạn sau.

Think→ Plan→ Build→ Review→ Test→ Ship→ Reflect

Smart Review Routing

gstack tự detect loại review phù hợp và bỏ qua những bước không cần thiết: design review bị skip cho backend-only changes, CEO review bị skip cho infra bug fixes. Các independent tasks được parallelize tự động.

Browser Tools

/browse — Real Browser Automation

Cung cấp Chromium được AI điều khiển:

  • ~100ms per command
  • Real clicks, screenshots, interactions
  • Anti-bot stealth built-in
  • Native support cho authenticated pages

Escape hatches:

  • Domain skills: Lưu per-site notes (ví dụ "Apply button in iframe") tự động fire khi revisit
  • Raw CDP: $B cdp <Domain.method> cho Chrome DevTools Protocol calls

/open-gstack-browser — Headed Browser Mode

Launch GStack Browser (custom Chromium build):

  • Anti-bot stealth và custom branding
  • Sidebar agent extension — natural language commands, routes to Sonnet (fast actions) hoặc Opus (reading/analysis), up to 5 minutes per task
  • One-click cookie import từ Chrome/Arc/Brave/Edge
  • Persistent session, không idle timeout
  • Real-world site compatibility (Google, NYTimes không bị CAPTCHA)

Prompt injection defense: 22MB ML classifier + Haiku transcript check + canary tokens + verdict combiner. Shield icon hiện status (green/amber/red). Opt-in 721MB DeBERTa ensemble qua GSTACK_SECURITY_ENSEMBLE=deberta.

Handoff: Gặp CAPTCHA/MFA? $B handoff mở Chrome visible ở cùng page với cookies/tabs. Solve xong, báo Claude, $B resume tiếp tục.

GBrain & Memory Systems

/setup-gbrain — Persistent Knowledge Base

Five-minute onboarding cho gbrain (persistent code memory). Bốn paths:

ModeMô tảSetup
Supabase existingPaste Session Pooler URL để kết nối~10s
Supabase auto-provisionPaste Personal Access Token; tự tạo project mới~90s
PGLite localZero accounts, zero network, isolated brain trên máy hiện tại~30s

Trust tiers per repo: read-write (search và write back), read-only (search only), deny (no access).

Keeping current: /sync-gbrain re-indexes repo code (incremental by default). Registers code search guidance trong CLAUDE.md.

/learn — Session Memory

Manage per-project learnings across sessions: review, search, prune patterns; export project-specific preferences; auto-decay 5%/week để giữ knowledge base không bị stale.

Parallel Development

gstack scale từ 1 sprint lên 10–15 sprint chạy đồng thời qua Conductor — nhiều Claude Code sessions trong isolated workspaces. Sprint structure ngăn chaos: mỗi agent biết chính xác phải làm gì và khi nào dừng. CEO reviews xử lý taste decisions; process xử lý determinism.

Checkpoint Mode

Opt-in, local by default:

gstack-config set checkpoint_mode continuous
  • Auto-commits work với prefix WIP: và structured [gstack-context] body (decisions, remaining, failed approaches)
  • Survives crashes — /context-restore reconstruct session từ WIP commits
  • /ship filter-squash WIP commits trước PR (giữ non-WIP cho clean bisect history)
  • Push opt-in: checkpoint_push=true (mặc định local-only để tránh trigger CI trên mọi WIP)

Multi-Agent Support

Setup auto-detects installed AI agents. gstack hoạt động với Claude Code, OpenAI Codex CLI, OpenCode, Cursor, Factory Droid, Kiro, Hermes, và GBrain mod.

Cross-Model Analysis: Chạy cả /review (Claude) và /codex (OpenAI) trên cùng branch để identify overlapping findings và highlight model-specific insights.

Configuration & Preferences

Command Prefix Options

Mặc định là short names (/qa, /review, /ship). Có thể chuyển sang namespaced (/gstack-qa, etc.):

# Short names (default)
cd ~/.claude/skills/gstack && ./setup --no-prefix

# Namespaced
cd ~/.claude/skills/gstack && ./setup --prefix

Auto-Upgrade

gstack-config set auto_upgrade true

Tự động upgrade khi first use mỗi session.

Privacy & Telemetry

Telemetry là opt-in (mặc định tắt). Nếu bật: gửi skill name, duration, success/fail, version, OS. Không bao giờ gửi: code, file paths, repo names, prompts, user content.

gstack-config set telemetry off
gstack-analytics  # local dashboard từ JSONL cá nhân

Philosophy & Approach

Builder Ethos

  • Boil the Ocean: Đừng assume narrow scope. /office-hours reframes vague requests thành complete problems.
  • Search Before Building: Đọc related code trước khi viết code mới. Match project style, conventions, libraries.
  • Three Layers of Knowledge: (1) Codebase patterns từ reading, (2) Project learnings từ /learn, (3) Design preferences từ /design-shotgun taste memory.

Gỡ cài đặt

# Automated
~/.claude/skills/gstack/bin/gstack-uninstall
# --keep-state để giữ config, --force để skip confirmation

# Manual: xóa browse daemons, per-skill dirs, gstack dir, integrations
pkill -f "gstack.*browse" 2>/dev/null || true
rm -rf ~/.claude/skills/gstack ~/.gstack
Lưu ý khi gỡ cài đặt thủ công: Cần xóa thêm các integration dirs (~/.codex/skills/gstack*, ~/.kiro/skills/gstack*...) và remove sections "## gstack" và "## Skill routing" khỏi CLAUDE.md trong các project.
