Tư duy cốt lõi: Tài liệu chính thức của Claude Code mô tả từng tính năng riêng lẻ nhưng không chỉ ra cách kết hợp chúng thành workflows thực tế. Hướng dẫn này lấp đầy khoảng trống đó bằng các visual tutorials, production-ready templates, và learning path có cấu trúc từ beginner đến advanced.

Claude Code Handbook là hướng dẫn có cấu trúc, example-driven để làm chủ Claude Code. Được tổ chức thành 10 modules với tổng thời gian học 11-13 giờ, tương thích với Claude Sonnet 4.6, Opus 4.8, và Haiku 4.5.

Module 1: Slash Commands

User-invoked shortcuts được lưu dưới dạng Markdown files trong .claude/commands/. Người dùng gọi thủ công khi cần.

  • Mỗi command là một file .md trong .claude/commands/
  • Ví dụ: optimize.md, pr.md, generate-api-docs.md
  • Gọi bằng cú pháp /tên-file trong Claude Code
# Cài đặt slash commands từ repo
cp 01-slash-commands/*.md .claude/commands/

Module 2: Memory

Persistent context duy trì thông tin xuyên suốt các sessions thông qua file CLAUDE.md.

  • Project-level — ./CLAUDE.md trong thư mục project
  • Directory-level — CLAUDE.md trong từng subdirectory
  • Personal-level — ~/.claude/CLAUDE.md cho preferences cá nhân
# Cài đặt project memory
cp 02-memory/project-CLAUDE.md ./CLAUDE.md
Best practice: Ghi lại trong CLAUDE.md: project context, coding conventions, thông tin team, và các quyết định kiến trúc quan trọng. Đây là "bộ nhớ dài hạn" của Claude Code cho project.

Module 5: Skills

Reusable, auto-invoked capabilities — khác với slash commands ở chỗ Skills được Claude tự động gọi khi phù hợp, không cần người dùng gọi thủ công.

  • Lưu tại ~/.claude/skills/
  • Mỗi skill có instructions và scripts đi kèm
  • Ví dụ: code-review-specialist — tự động kích hoạt khi Claude review code
# Cài đặt skill
cp -r 03-skills/code-review-specialist ~/.claude/skills/

Module 8: Subagents

Specialized AI assistants với isolated contexts, mỗi subagent chuyên về một nhiệm vụ cụ thể.

  • Lưu tại .claude/agents/
  • Mỗi agent có context riêng biệt, không bị ảnh hưởng bởi conversation chính
  • Use cases: code review, testing, documentation, security analysis
  • Hỗ trợ parallel execution — nhiều agents chạy đồng thời
Tư duy subagents: Thay vì để một Claude làm tất cả, decompose công việc thành các specialists. Mỗi subagent nhận task, thực hiện trong isolated context, và trả kết quả về orchestrator.

Module 6: Hooks

Event-driven shell commands tự động chạy khi các sự kiện nhất định xảy ra trong Claude Code. Tổng cộng có 29 events trên 5 hook types.

Hook TypeKhi nào kích hoạt
PreToolUseTrước khi Claude dùng một tool
PostToolUseSau khi Claude dùng một tool
SessionStartKhi bắt đầu session mới
TaskCompletedKhi task hoàn thành
(và 1 type khác)Xem tài liệu hook types đầy đủ
  • Hook scripts lưu tại ~/.claude/hooks/
  • Dùng để: auto-format code, run linters, log actions, trigger CI/CD

Module 7: MCP Protocol

Model Context Protocol (MCP) cho phép Claude Code kết nối với external tools và services.

  • Truy cập GitHub, databases, filesystems qua giao thức chuẩn
  • Cấu hình qua environment variables và CLI setup
  • Cho phép Claude đọc/ghi files ngoài project, query databases, gọi APIs bên ngoài
Lưu ý bảo mật: MCP servers chạy với permissions của user. Chỉ cài đặt MCP servers từ nguồn đáng tin cậy và kiểm tra kỹ permissions trước khi enable.

Module 10: Plugins

Bundled feature collections — đóng gói nhiều components lại với nhau thành một gói có thể cài đặt một lần.

  • Một plugin có thể bao gồm: commands, agents, MCP configs, và hooks
  • Ví dụ: pr-review plugin, devops-automation plugin
  • Cài đặt bằng /plugin install command

Module 3: Checkpoints

Session snapshots cho phép lưu trạng thái hiện tại và rewind về điểm trước đó.

  • Tạo checkpoint tại bất kỳ thời điểm nào trong session
  • Rewind để thử các cách tiếp cận khác nhau từ cùng một điểm
  • Hữu ích khi thử nghiệm nhiều solutions cho cùng một vấn đề

Module 4: CLI Reference

Các terminal commands cho Claude Code:

  • Interactive mode — Giao diện tương tác mặc định
  • Print mode — Output trực tiếp ra terminal, không cần interaction
  • File processing — Xử lý files như input
  • JSON output — Structured output cho programmatic use
  • Session resumption — Tiếp tục session đã dừng trước đó
CI/CD Integration: CLI Reference module đặc biệt hữu ích cho việc tích hợp Claude Code vào CI/CD pipelines, sử dụng headless mode và JSON output để automate code review, documentation generation, và security scanning.

Real-World Workflows

Handbook trình bày cách kết hợp các tính năng để giải quyết các use cases thực tế:

Automated Code Review

Slash Commands→ Subagents→ Memory→ MCP

Trigger review qua slash command, subagent chuyên biệt thực hiện, memory giữ context về coding standards, MCP truy cập GitHub để tạo PR comments.

Documentation Generation

Skills→ Subagents→ Memory

Skills tự động nhận diện code cần document, subagents generate docs cho từng component, memory duy trì style guide nhất quán.

DevOps Deployment

Plugins→ MCP→ Hooks

Plugin cung cấp deployment commands, MCP kết nối cloud services, hooks trigger notifications và rollback khi cần.

Các workflows khác được đề cập

  • Team onboarding automation
  • Security audit pipeline
  • Complex refactoring với checkpoints
