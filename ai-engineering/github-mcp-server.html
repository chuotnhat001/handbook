GitHub MCP Server

GitHub's official Model Context Protocol server — kết nối AI tools trực tiếp với GitHub platform qua natural language

Nguồn: github.com/github/github-mcp-server — by GitHub, MIT License

GitHub MCP Server là implementation chính thức của GitHub cho Model Context Protocol (MCP). Server này cho phép AI agents và assistants tương tác trực tiếp với repositories, quản lý issues và pull requests, phân tích code, và tự động hóa workflows thông qua ngôn ngữ tự nhiên.

Tư duy cốt lõi: Thay vì AI chỉ đọc code, GitHub MCP Server cho phép AI agents hành động trực tiếp trên GitHub — tạo PR, quản lý issues, phân tích CI/CD, kiểm tra security alerts — tất cả qua một giao diện MCP chuẩn hóa với hơn 100 tools sẵn có.

Remote Setup

Remote setup là cách đơn giản nhất — không cần Docker hay cài đặt thêm. GitHub host server tại https://api.githubcopilot.com/mcp/.

  • Hỗ trợ one-click installation cho VS Code 1.101+, VS Code Insiders, và Visual Studio
  • Authentication qua OAuth (browser-based login)
  • Hoặc dùng Personal Access Token (PAT)
Khuyến nghị: Với hầu hết người dùng cá nhân, Remote Setup + OAuth là lựa chọn nhanh nhất để bắt đầu.

Local Setup (Docker)

Chạy server local bằng Docker image chính thức:

docker run -i --rm \
  -p 127.0.0.1:8085:8085 \
  -e GITHUB_OAUTH_CALLBACK_PORT=8085 \
  ghcr.io/github/github-mcp-server

Port mặc định cho OAuth callback là 127.0.0.1:8085. Có thể thay đổi qua biến môi trường GITHUB_OAUTH_CALLBACK_PORT.

Build from Source

Yêu cầu Go. Build ra binary rồi cấu hình thủ công trong MCP host settings:

# Clone và build
git clone https://github.com/github/github-mcp-server
cd github-mcp-server
go build -o github-mcp-server ./cmd/github-mcp-server

Toolsets

Server tổ chức functionality thành khoảng 20 toolsets. Default toolset bao gồm: context, repos, issues, pull_requests, users.

Cấu hình toolsets qua environment variable hoặc CLI flag:

# Chọn toolsets cụ thể
GITHUB_TOOLSETS="repos,issues,pull_requests,actions"

# Bật tất cả toolsets
--toolsets all

Core Features

  • Repository Management — Browse code, search files, analyze commits, hiểu project structure
  • Issue & PR Automation — Tạo, cập nhật, và quản lý issues và pull requests với AI assistance
  • CI/CD Intelligence — Monitor GitHub Actions workflows, phân tích build failures, quản lý releases
  • Code Analysis — Kiểm tra security findings, review Dependabot alerts, phân tích code patterns
  • Team Collaboration — Truy cập discussions, quản lý notifications, streamline team processes

Insiders Mode

Insiders Mode cung cấp early access tới experimental tools và features chưa được release chính thức:

# Bật insiders mode
github-mcp-server --insiders
Lưu ý: Insiders features là experimental — có thể thay đổi hoặc bị loại bỏ bất kỳ lúc nào. Không nên dùng trong production workflows quan trọng.

Enterprise Support

GitHub MCP Server hỗ trợ đầy đủ GitHub Enterprise:

  • GitHub Enterprise Server (GHES) — Trỏ tới on-premise instance qua GITHUB_HOST
  • GitHub Enterprise Cloud with Data Residency — Hỗ trợ ghe.com domains
# GitHub Enterprise Server
export GITHUB_HOST="github.your-company.com"

# GitHub Enterprise Cloud với data residency
export GITHUB_HOST="your-org.ghe.com"

Token Security

GitHub repo nhấn mạnh các best practices sau khi dùng PAT:

  • Lưu PAT trong environment variables — không hardcode vào source code
  • Thêm file chứa token vào .gitignore
  • Dùng minimum required scopes: repo, read:packages, read:org
  • Rotate tokens định kỳ
  • Hạn chế file permissions trên các file chứa credentials
Anti-pattern: Không bao giờ commit PAT hoặc OAuth token vào Git history. Dùng --read-only flag khi chỉ cần đọc dữ liệu để giảm thiểu rủi ro nếu token bị lộ.
