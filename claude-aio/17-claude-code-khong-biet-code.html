Claude Code Cho Người Không Biết Code

Vibecoding — xây ứng dụng web bằng cách nói chuyện với AI, không cần biết lập trình

Nguồn: phandonggiang.com — Cấp độ 4, Chuyên sâu
Tư duy cốt lõi: Vibecoding = xây ứng dụng web bằng ngôn ngữ đời thường. Không thay thế developer — mà tạo mockup nhanh, công cụ nội bộ, prototype cá nhân. Cảnh báo: "Năng suất trình diễn" — đừng xây thứ bạn sẽ không dùng.

Hai use case chính

  • Tạo mockup có thể click để trình bày với developer/designer
  • Xây công cụ cá nhân hoặc cho team mà không cần polish chuyên nghiệp

Cài đặt (7 bước)

  1. Tải Claude app từ claude.com/download, đăng ký Pro ($20/tháng)
  2. Tạo thư mục làm việc sạch — không chứa mật khẩu, file riêng tư
  3. Bật "Allow bypass permissions mode" trong Settings
  4. Kết nối Netlify và Supabase làm connectors
  5. Chọn thư mục làm việc
  6. Chọn model Opus 4.8 (High)
  7. Sẵn sàng xây dựng

Mega Prompt — System Prompt Mẫu

Cấu trúc: giữ nguyên framework, chỉ thay phần mô tả mục tiêu. Các yếu tố:

  • Đặt Claude vào vai CTO; người dùng là CEO không biết code
  • Dùng ngôn ngữ đơn giản, tránh thuật ngữ kỹ thuật
  • Claude phải phỏng vấn người dùng bằng AskUserQuestion trước khi build
  • Dùng Netlify để deploy, Supabase để lưu data
  • Build theo kiểu incremental — từng tính năng một
  • Test từng section trước khi show cho người dùng

Kỹ năng vibecoding tốt hơn

Điểm khởi đầu

  • Bắt đầu với ảnh tham khảo thiết kế bạn thích
  • Mô tả kết quả mong muốn theo ngôn ngữ đời thường
  • Build incremental, không build tất cả một lúc
  • Feedback loop: chụp màn hình vấn đề → liệt kê lỗi → lặp lại

Chi phí và quản lý token

Claude Code tiêu token nhanh hơn chat — vì mỗi tin nhắn gửi kèm toàn bộ lịch sử + files.

Khuyến nghị

  • Mở conversation mới cho mỗi task
  • Tier $20: dùng Sonnet làm mặc định
  • Tier $100: mới dùng Opus 4.8 High
  • Trỏ Claude đến file có sẵn thay vì paste text lớn
  • Yêu cầu Claude lập kế hoạch trước khi thay đổi lớn
  • Theo dõi usage: gõ /usage trong Claude Code

Bàn giao cho Developer (HANDOFF.md)

Khi prototype đủ tốt để bàn giao cho dev thật, tạo file HANDOFF.md gồm:

  • Mô tả app làm gì, ai dùng
  • Phần đã hoàn chỉnh vs phần chưa xong
  • Tech stack đã dùng (Netlify, Supabase)
  • Thông tin setup quan trọng
  • Ghi rõ: đây là prototype, không phải production code
  • Giữ ngắn gọn — khoảng 5 phút đọc
