Thiết Lập Claude Cowork Đúng Cách

Hướng dẫn thực chiến từ Phan Đồng Giang — cấu trúc folder, file context, Global Instructions, Wispr Flow và 6 cách tiết kiệm token

Nguồn: phandonggiang.com — Khóa học AI thực chiến
Tư duy cốt lõi: Claude Cowork hoạt động tốt nhất khi bạn đầu tư 20 phút thiết lập đúng ngay từ đầu — ba file context gọn (dưới 6.000 token), Global Instructions rõ ràng, và dùng giọng nói thay vì gõ để giữ ngữ cảnh tự nhiên.

Cấu trúc thư mục Cowork

Bộ ba thư mục chính tạo nên nền tảng làm việc với Claude Cowork:

Thư mụcNội dungMục đích
ABOUT ME/3 file mô tả về người dùngClaude đọc trước mỗi tác vụ để hiểu ngữ cảnh
OUTPUTS/Kết quả công việc của ClaudeLưu trữ theo từng project con
TEMPLATES/Sản phẩm tốt nhất đã được lược bỏ nội dung cụ thểTái sử dụng khung sườn cho lần sau

Ba file cốt lõi trong ABOUT ME

Đây là phần quan trọng nhất. Ba file này giúp Claude hiểu bạn là ai và làm việc theo đúng phong cách của bạn.

File 1: about-me.md

Mô tả bạn là ai, cách bạn suy nghĩ, phong cách viết yêu thích. Giữ dưới 2.000 từ để tiết kiệm context window. Nội dung cần bao gồm:

  • Vai trò, công ty, ngành nghề
  • Công cụ dùng hằng ngày, quy trình làm việc
  • Tiêu chí chất lượng công việc
  • Thói quen hoặc lĩnh vực thường sai
  • Nguyên tắc cốt lõi bạn tuân theo
  • Quan điểm cá nhân khác biệt

File 2: anti-ai-writing-style.md

Liệt kê các từ, cấu trúc câu bạn ghét để Claude tránh dùng. Bao gồm:

  • Danh sách từ bị cấm — ví dụ: hơn 80 từ kiểu "delve, harness, tapestry"
  • Quy tắc định dạng ưa thích
  • Các cấu trúc câu nghe giả tạo hoặc quá "AI"

File 3: my-company.md

Dưới 1.000 từ. Chỉ ba mục:

  • Mục tiêu (của công ty hoặc dự án)
  • Đang tập trung vào gì trong quý này
  • Đang từ chối những gì
Giới hạn tổng cộng: Ba file cộng lại không nên vượt quá 6.000 token. Claude đọc toàn bộ trước mỗi tác vụ — file quá lớn đồng nghĩa với hàng nghìn token lãng phí mỗi lần.

Thiết lập Global Instructions

Đây giống như một prompt cố định mà Claude Cowork luôn đọc trước khi làm bất cứ việc gì.

Truy cập qua: Settings → Cowork → Edit Global Instructions. Các nguyên tắc cần cấu hình:

  • Claude phải đọc hết các file trong ABOUT ME/ trước mỗi tác vụ
  • Không bao giờ đọc OUTPUTS/ hay TEMPLATES/ trừ khi được chỉ định rõ
  • Lưu mọi sản phẩm vào OUTPUTS/ trong folder con theo tên project
  • Dùng AskUserQuestion thay vì tự đoán khi thiếu thông tin
Mẹo: Sau khi dán Global Instructions, chỉnh lại tên file cho khớp chính xác với tên folder và file bạn đã tạo trên máy.

Cách Templates hoạt động

Templates giúp tái sử dụng khung sườn của những sản phẩm tốt mà không cần tạo lại từ đầu.

Tạo sản phẩm tốt → Yêu cầu lưu template → Claude lược bỏ nội dung cụ thể → Dùng lại lần sau

Khi Cowork tạo ra sản phẩm bạn thích, chỉ cần nói: "Lưu kết quả này thành template". Claude sẽ giữ lại khung sườn và cấu trúc, loại bỏ nội dung cụ thể.

Lần sau muốn dùng lại: "Dùng template trong TEMPLATES/[tên-file]".

Wispr Flow — Nói thay vì gõ

Wispr Flow là ứng dụng chuyển giọng nói thành văn bản trực tiếp vào bất kỳ ô nhập liệu nào — bao gồm chat với Claude.

Cách nhậpTốc độChất lượng ngữ cảnh
Gõ bàn phím~60 từ/phútThường ngắn gọn, thiếu chi tiết
Nói với Wispr Flow~150 từ/phútTự nhiên hơn, đúng giọng của bạn hơn
"Não bạn suy nghĩ khác hẳn khi phải nói ra thành lời."

Cài đặt Wispr Flow

  1. Vào wispr.ai, tải và cài đặt ứng dụng
  2. Chọn phím tắt yêu thích (ví dụ: phím Shift)
  3. Giữ phím và nói, thả ra — chữ tự động xuất hiện vào ô đang focus

Cách sử dụng hiệu quả

  • Prompt ban đầu: nói ra thay vì gõ để truyền đủ ngữ cảnh
  • Trả lời AskUserQuestion: click lựa chọn hoặc nói thêm để bổ sung ngữ cảnh
  • Phản hồi điều chỉnh: mô tả bằng lời thay vì tự chỉnh sửa trực tiếp

Tiết kiệm Credit (Token)

Claude tính phí theo token — mỗi tin nhắn phải đọc lại toàn bộ lịch sử hội thoại. Hiểu cơ chế này giúp giảm chi phí đáng kể.

Token tích lũy nhanh hơn bạn nghĩ: 20 tin nhắn = ~105.000 token. 30 tin nhắn = ~232.000 token. Mỗi tin nhắn thêm vào buộc Claude đọc lại toàn bộ phía trên.

6 cách giảm token tiêu thụ

  1. Restart cuộc trò chuyện thay vì gửi tin nhắn tiếp
    Bấm nút restart ở vị trí cao hơn trong lịch sử thay vì tiếp tục cuộc trò chuyện dài.
  2. Bắt đầu phiên mới sau ~20 tin nhắn
    Yêu cầu Claude tóm tắt toàn bộ → mở phiên mới → dán tóm tắt làm tin nhắn đầu tiên.
  3. Gộp nhiều việc vào một tin nhắn
    3 prompt riêng = 3 lần tải ngữ cảnh. 1 prompt gộp = 1 lần tải.
  4. Dùng Sonnet cho việc đơn giản
    Kiểm tra ngữ pháp, brainstorm, định dạng — chi phí chỉ bằng phần nhỏ so với Opus. Để Opus + Extended Thinking cho việc phức tạp.
  5. Giữ file ABOUT ME gọn nhẹ
    Cowork đọc trước mỗi tác vụ. Target: dưới 6.000 token tổng cộng 3 file.
  6. Dàn trải công việc trong ngày
    Claude dùng khung giới hạn 5 giờ xoay vòng. Chia thành 2–3 phiên: sáng, chiều, tối. Tránh khung cao điểm (5–11h sáng giờ Thái Bình Dương).

Thiết lập trong 20 phút

Toàn bộ quá trình từ zero đến sẵn sàng làm việc chỉ mất 20 phút.

Thời gianViệc cần làm
Phút 0–5Tải Claude Desktop, đăng ký gói trả phí. Tạo cấu trúc: ABOUT ME/ (3 file) + OUTPUTS/ + TEMPLATES/
Phút 5–6Dán Global Instructions vào Settings → Cowork → Edit Global Instructions. Chỉnh tên file khớp với folder.
Phút 6–8Vào wispr.ai, tải và cài đặt Wispr Flow. Chọn phím tắt.
Phút 8–15Chạy phiên làm việc đầu tiên bằng giọng nói. Trả lời câu hỏi bằng giọng nói.
Phút 15–20Yêu cầu tạo template từ cuộc trò chuyện. Kiểm tra lại từng folder trên máy.
Kết quả sau 20 phút: Claude biết bạn là ai, biết lưu kết quả đúng chỗ, không tự đoán khi thiếu thông tin, và bạn có một workflow nhập liệu nhanh gấp 2,5 lần so với gõ tay.
